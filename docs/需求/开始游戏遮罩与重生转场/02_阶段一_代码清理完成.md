## 关键改动说明

### 1. 重复代码消除（DRY 原则）

**问题**：`formatTime()` 函数在 `JumpGame.js` 和 `PlayerBestStore.js` 中重复定义

**解决方案**：
- 创建 `src/utils/timeUtils.js`，导出通用的 `formatTime(ms)` 函数
- `PlayerBestStore.js` 直接导入并使用
- `JumpGame.js` 保持 `formatTime()` 实例方法作为代理，内部调用工具函数，确保向后兼容

```javascript
// src/utils/timeUtils.js
export function formatTime(ms) {
  const totalSeconds = Math.floor(ms / 1000)
  const minutes = Math.floor(totalSeconds / 60)
  const seconds = totalSeconds % 60
  return `${minutes.toString().padStart(2, '0')}:${seconds.toString().padStart(2, '0')}`
}

// PlayerBestStore.js
import { formatTime } from '@utils/timeUtils.js'
getFormatted() {
  if (this.bestTime === Infinity) return '--:--'
  return formatTime(this.bestTime)  // 使用工具函数
}

// JumpGame.js（保持向后兼容）
import { formatTime as formatTimeUtil } from '@utils/timeUtils.js'
formatTime(ms) {
  return formatTimeUtil(ms)  // 代理到工具函数
}
```

### 2. 内存泄漏修复

**问题**：`NetworkView.js` 和 `HistoryView.js` 每次创建都添加 resize 监听器，但从未移除

**解决方案**：
- 在构造函数中绑定 resize 处理函数为实例属性 `_resizeHandler`
- 在 `destroy()` 方法中调用 `removeEventListener` 移除监听器
- 同时清理 canvas DOM 元素

```javascript
constructor(containerId) {
  // ...
  // 绑定 resize 处理函数以便后续移除
  this._resizeHandler = () => this.resize()
  this.init()
}

init() {
  // ...
  window.addEventListener('resize', this._resizeHandler)
}

destroy() {
  // 清理 resize 监听器
  window.removeEventListener('resize', this._resizeHandler)
  this.canvas?.remove()
}
```

### 3. 统一属性访问方式

**问题**：`main.js` 中混用 `game.generation` 和 `game.getState().generation`

**解决方案**：
- 统一使用 `game.getState().generation` 访问 generation 属性
- 修改位置：`recordResult()` 函数和 `updateGameInfo()` 函数
