# 阶段三：重生转场效果 - 完成说明

## 完成时间
2026-03-30

## 实现内容

### 1. 新增 TRANSITIONING 状态
**文件**: `src/game/JumpGame.js`

在 STATUS 常量中新增了 `TRANSITIONING` 状态，用于标识转场期间：
```javascript
export const STATUS = {
  READY: 'ready',
  IDLE: 'idle',
  MOVING: 'moving',
  DEAD: 'dead',
  WON: 'won',
  TRANSITIONING: 'transitioning'  // 新增：转场中
}
```

### 2. 新建转场管理器
**文件**: `src/render/TransitionManager.js` (新增)

创建了专门的转场管理器，处理死亡/胜利后的渐暗渐亮效果：
- 等待缓冲时间（800ms）
- 渐暗（400ms）
- 执行重生逻辑（在暗屏时）
- 短暂停顿（100ms）
- 渐亮（400ms）

### 3. 重构重生逻辑
**文件**: `src/game/JumpGame.js`

修改了 `triggerDeath()` 和 `triggerWin()` 方法：
- 如果存在 `onTransitionStart` 回调，使用转场方式重生
- 否则保持兼容，直接重生（旧行为）

新增辅助方法：
- `_executeRespawn()`: 在暗屏时执行重生
- `_onRespawnComplete()`: 转场完成后的处理

新增转场回调：
- `onTransitionStart`: 转场开始
- `onTransitionEnd`: 转场结束

### 4. 集成转场管理器
**文件**: `src/main.js`

- 导入并创建 `TransitionManager` 实例
- 绑定 `onTransitionStart` 回调
- 绑定 `onTransitionEnd` 回调：
  - 玩家模式：显示开始遮罩，重置计时器
  - AI 模式：自动开始新世代
- 修改 `onGenerationChange`：不再在此显示遮罩或启动计时器
- HMR 清理：添加 `transitionManager.destroy()`

## 转场时序流程

```
玩家死亡/胜利
    ↓
triggerDeath()/triggerWin() 被调用
    ↓
执行 onDeath/onWin 回调（显示死亡/胜利动画）
    ↓
onTransitionStart 被调用
    ↓
等待 800ms（展示死亡/胜利结果）
    ↓
渐暗开始（400ms）
    ↓
渐暗完成 → 调用 onMidPoint 回调
    ↓
_executeRespawn() 执行（暗屏中）：
  - generation++
  - init() 重置世界
  - status = TRANSITIONING
    ↓
等待 100ms（确保重置完成）
    ↓
渐亮开始（400ms）
    ↓
渐亮完成 → 调用 onComplete 回调
    ↓
onTransitionEnd 被调用：
  - 玩家模式：显示开始遮罩，重置计时器
  - AI 模式：自动开始
    ↓
游戏继续
```

总转场时长：约 1.7 秒
