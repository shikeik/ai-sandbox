# 开始游戏遮罩与重生转场

> 分支：`feature/game-flow-enhancement`  
> 创建日期：2026-03-30

---

## 需求概述

改善游戏的初始体验和重生体验，增加遮罩层控制和转场动画。

---

## 需求详情

### 1. 开始游戏遮罩层

#### 触发时机
页面加载完成（DOMContentLoaded），游戏初始化后显示遮罩。

#### 视觉表现
- **背景**：半透明黑色遮罩（rgba(0,0,0,0.7)）覆盖整个游戏区域
- **内容**：中央显示"开始游戏"按钮
- **按钮样式**：醒目设计，支持点击/触摸

#### 交互逻辑
```
页面加载 → 游戏初始化 → 显示遮罩 → 等待玩家点击
                                           ↓
点击按钮 → 遮罩渐隐消失 → 游戏正式开始 → 启动计时器
```

#### 状态变更
- 新增游戏状态：`READY`（准备中）、`PLAYING`（进行中）
- 当前状态：`IDLE`、`MOVING`、`DEAD`、`WON` 需与新增状态兼容

#### 计时器规则
- **未点击开始**：不启动计时器
- **点击开始后**：正式开始计时
- **重生后**：计时器重置，重新开始计时

---

### 2. 重生转场效果

#### 触发时机
玩家死亡（`onDeath`）或胜利（`onWin`）后。

#### 转场流程
```
死亡/胜利动画完成
        ↓
   等待缓冲（800ms）
        ↓
   屏幕渐暗（fade out，400ms）
        ↓
   重生/生成新关卡
        ↓
   屏幕渐亮（fade in，400ms）
        ↓
   游戏继续
```

#### 视觉表现
- **渐暗**：遮罩层透明度从 0 → 1
- **渐亮**：遮罩层透明度从 1 → 0
- **缓动函数**：ease-in-out，平滑过渡

#### 与现有逻辑的差异

| 阶段 | 现状 | 新需求 |
|------|------|--------|
| 死亡/胜利后 | 直接调用 `_nextGeneration()` | 等待 → 渐暗 → 重生 → 渐亮 |
| 玩家可见性 | 跳变，世界突然变化 | 平滑过渡，有转场感知 |
| 计时器 | 立即重置并开始 | 转场完成后重置并开始 |

## 技术要点

### 涉及文件
- `src/main.js` - 主流程控制、遮罩管理
- `src/game/JumpGame.js` - 游戏状态扩展
- `src/render/GameRenderer.js` - 视觉渲染、动画
- `src/style.css` - 遮罩层样式

### 新增状态常量
```javascript
export const STATUS = {
  READY: 'ready',      // 新增：等待开始
  PLAYING: 'playing',  // 新增：游戏进行中（可能合并到 IDLE）
  IDLE: 'idle',
  MOVING: 'moving',
  DEAD: 'dead',
  WON: 'won'
}
```

### 遮罩层 DOM 结构建议
```html
<!-- 开始游戏遮罩 -->
<div id="start-overlay" class="game-overlay">
  <button id="start-btn">开始游戏</button>
</div>

<!-- 转场遮罩（可复用） -->
<div id="transition-overlay" class="transition-overlay"></div>
```

### 时序控制
需要协调多个异步操作：
1. 动画完成回调（死亡/胜利动画）
2. setTimeout 延迟
3. CSS transition 过渡
4. 游戏逻辑重置

## 风险评估

### 高风险点
1. **状态管理复杂化**：新增 READY 状态可能影响现有状态机
2. **时序竞态**：转场动画与游戏逻辑重置的时序需要精确控制
3. **计时器逻辑变更**：计时启动点从 `onGenerationChange` 改为"开始按钮点击"

### 中风险点
1. **CSS 层级管理**：遮罩层 z-index 需要正确设置（高于游戏画面，低于菜单）
2. **触摸/键盘事件**：开始遮罩显示期间需要禁用游戏控制
3. **热更新兼容性**：Vite HMR 可能需要处理遮罩状态

### 低风险点
1. **样式细节**：按钮样式、遮罩透明度可微调

## 验收标准

- [ ] 胜利后有渐暗渐亮转场
- [ ] 转场期间游戏不接受输入
- [ ] 转场完成后计时器正确重置
