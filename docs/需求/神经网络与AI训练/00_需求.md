> 创建时间：2026-03-30

## 一、背景

当前 `feature/neural-network` 分支已实现基础神经网络结构和可视化框架，但存在以下问题：

1. **核心逻辑与需求不符** - 决策使用Softmax而非纯贪心，训练按输入加权而非固定值
2. **可视化不完善** - 权重数值显示条件苛刻，缺少权重矩阵展示
3. **AI训练未实现** - 切换到AI模式后无法自动运行，缺少单步/自动训练控制

---

## 二、核心需求

### 2.1 神经网络核心逻辑

#### 2.1.1 决策逻辑（decide）

| 项目 | 需求 |
|------|------|
| **方法** | 纯贪心比较得分 |
| **输入** | 3个格子状态（坑=1，平地=0） |
| **计算** | `移动得分 = grid1×w11 + grid2×w12 + grid3×w13` |
| **决策** | 比较两个得分，选高的：`jumpScore > moveScore ? 1 : 0` |
| **输出** | 0=移动，1=跳跃 |

**禁止**：使用Softmax转概率（当前代码问题）

#### 2.1.2 训练逻辑（train）

| 场景 | 更新规则 |
|------|----------|
| **正确时**（存活+0.02，胜利+1） | 选中动作的权重 += `learningRate`（固定0.2） |
| **错误时**（死亡-1） | 选中动作的权重 -= `learningRate`（固定0.2）<br>其他动作权重 += `learningRate / 2`（0.1） |

**注意**：权重更新是固定数值，**不乘以输入值**（当前代码问题）

#### 2.1.3 网络结构配置

```javascript
{
  layerSizes: [3, 2],      // 3输入 → 2输出，单层感知机
  learningRate: 0.2,       // 学习率可配置（默认0.2）
  weightClip: 5,           // 权重裁剪范围 [-5, 5]
  useBias: false           // 不使用偏置（预留但默认关闭）
}
```

### 2.3 AI训练控制

#### 2.3.1 模式切换

菜单提供三种模式：

| 模式 | 行为 |
|------|------|
| **玩家游玩** | 人工操作，AI不干预 |
| **AI控制** | AI自动决策，每步之间等待固定时间 |
| **AI训练** | AI自动循环，死亡/胜利后自动重生继续训练 |

#### 2.3.2 自动启动

- 切换到AI模式后，**自动开始新一局**（不需要手动点"开始游戏"）
- 死亡/胜利后自动重生，继续训练

#### 2.3.3 速度控制

在AI模式下，提供速度选择：

| 档位 | 延迟 | 适用场景 |
|------|------|----------|
| **单步** | 每步等待人工确认 | 调试、观察决策 |
| **慢速** | 1秒/步 | 观察AI行为 |
| **中速** | 200ms/步 | 正常训练 |
| **快速** | 50ms/步 | 批量训练 |
| **极速** | 无延迟 | 最快速训练 |

**UI实现**：在菜单中添加速度选择子菜单或滑块

#### 2.3.4 单步执行

- 单步模式下，AI决策后暂停
- 提供"下一步"按钮或手势触发下一步
- 当前状态（输入、得分、权重变化）实时显示

## 三、代码质量要求

参考代码重构规范，满足：

| 原则 | 要求 |
|------|------|
| **DRY** | 输入转换、训练触发等逻辑不重复 |
| **SRP** | 决策、训练、渲染分离，一个函数只做一件事 |
| **配置化** | 学习率、速度等参数可配置，禁止魔法数字 |
| **扩展性** | 保留隐藏层扩展能力，但当前保持单层 |

## 四、验收标准

### 4.1 核心逻辑
