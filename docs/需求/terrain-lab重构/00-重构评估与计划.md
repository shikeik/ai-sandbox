# terrain-lab 代码重构需求文档

> 编写时间：2026-04-04
> 评估范围：`src/terrain-lab/` 全部 TypeScript 源码
> 当前问题：`main.ts` 长达 1523 行，是典型的 God File，职责混乱、逻辑与视图重度耦合
> 目标：降低代码复杂度，实现逻辑与视图分离，提取内联类

---

## 一、项目背景

`terrain-lab` 是 AI 神经元训练沙盘的第二个核心模块（监督/无监督学习演示页）。随着功能迭代，代码文件 `main.ts` 已经膨胀到 1523 行，同时承担了：
- 神经网络训练算法（监督/无监督）
- 课程学习控制
- 4 种 Canvas 绘制（编辑器、MLP 网络图、Embedding 空间、执念曲线）
- UI 更新（DOM 操作）
- 动画系统
- 事件绑定与全局函数暴露

这导致：
- 任何小改动都要在 1500+ 行里找位置
- 训练逻辑无法脱离浏览器环境测试
- 课程学习的监督/无监督代码有 80% 重复
- Canvas DPR 初始化代码复制了 5 遍

## 二、代码异味详细清单

### 2.1 DRY 原则违反（重复代码）

| 异味 | 位置 | 说明 |
|---|---|---|
| Canvas DPR 初始化复制 5 遍 | `main.ts` 的 `drawEditorWithState` / `drawMLP` / `drawEmbedding` / `drawObsessionCurve` / `stepAnimation` | 完全相同的 `dpr` 获取、`canvas.width/height` 设置、`setTransform`、 `clearRect` |
| 课程学习监督/无监督训练逻辑 80% 雷同 | `runCurriculumSupervised()` vs `runCurriculumUnsupervised()` | while 循环结构、评估、快照保存、UI 更新完全一致，只有梯度累积方式不同 |
| `GradientBuffer` 接口重复定义 | `supervised.ts` 和 `unsupervised.ts` | 两个文件中 `GradientBuffer`、`isValidGradientBuffer`、`getTotalGradientMagnitude` 完全一样 |
| 地形生成核心逻辑重复 | `terrain.ts` 的 `generateTerrainData()` 和 `generateRandomTerrain()` | 随机地形生成、清理多余狐狸、放置狐狸的代码块完全复制 |

### 2.2 SRP 原则违反（过长函数与 God File）

| 函数/文件 | 行数 | 职责过多表现 |
|---|---|---|
| `main.ts` | 1523 | 同时承载训练算法、课程学习、4 种 Canvas 绘制、动画、UI 更新、事件绑定 |
| `trainUnsupervised()` | 103 | ε-贪心采样、奖励计算、过滤式更新、探索率调整全在一个函数 |
| `drawMLP()` | 106 | 坐标计算、连线绘制、节点绘制、文字标注混在一起 |
| `drawObsessionCurve()` | 98 | 坐标轴、数据线、指示线、刻度、图例全在一个函数 |
| `runCurriculumUnsupervised()` | 80 | 训练循环 + 评估 + UI 更新 |
| `validateTerrain()` | 78 | 4 个动作的详细 HTML 报告拼接逻辑冗长 |
| `init()` | 94 | 事件绑定、全局函数暴露、ResizeObserver、控制台初始化混在一起 |

### 2.3 逻辑与视图重度耦合（Logic-View Coupling）

以下函数在执行业务逻辑时**直接操作 DOM 或 Canvas**：

| 函数 | 耦合点 | 后果 |
|---|---|---|
| `trainSupervised()` / `trainUnsupervised()` | 训练循环内部直接调用 `updateMetrics()` | 训练算法无法脱离浏览器测试 |
| `runCurriculumSupervised()` / `runCurriculumUnsupervised()` | 直接调用 `updateMetrics`、`updateSnapshotSlider`、`drawObsessionCurve`、`updateExam` | 课程控制流和视图流混在一起 |
| `predict()` | forward 后立刻顺序调用 `updateTerrainStatus`、`drawMLP`、`drawEmbedding`、`updateProbs`、`playAnimation` | 无法单独测试预测逻辑正确性 |
| `validateTerrain()` | 规则判定后直接拼接 HTML 字符串 | 判定逻辑和报告格式绑死 |
| `generateData()` / `randomTerrain()` / `resetNet()` | 直接操作 DOM 元素更新显示 | 数据层和视图层无分界线 |
| `recordSnapshotStats()` | 遍历数据集计算 acc/loss，却放在 `main.ts` 中像 UI helper | 重计算逻辑位置错误 |

## 三、重构目标

### 总体目标
1. **`main.ts` 压缩到 < 250 行**：只保留初始化、事件绑定、全局函数暴露
2. **逻辑与视图完全分离**：训练算法、规则判定等纯逻辑函数不碰 DOM
3. **提取 6 个内联类**：把紧密相关的函数和数据封装成独立模块
4. **消除所有高优先级重复代码**

---

## 四、重构计划（分 7 批执行）

### 批次 1：提取 Canvas DPR 工具函数
- **改动文件**：`renderer.ts`、`main.ts`
- **内容**：把重复的 DPR 初始化代码提取为 `setupCanvas(canvas): { ctx, rect }`
- **收益**：消灭 5 处重复，减少 ~40 行

### 批次 2：Canvas 绘制独立化
- **改动文件**：`renderer.ts`、`main.ts`
- **内容**：把 `drawMLP`、`drawEmbedding`、`drawObsessionCurve`、`drawEditorWithState`、`drawEditorLabels` 从 `main.ts` 全部迁入 `renderer.ts`
- **收益**：`main.ts` 减少 ~350 行，渲染逻辑集中管理

### 批次 3：UI 管理器独立化
- **改动文件**：新建 `ui-manager.ts`、`main.ts`
- **内容**：提取 `UIManager` 类，收口所有 DOM 更新（`updateMetrics`、`updateExam`、`updateTerrainStatus`、`updateProbs`、`updateObsessionStatus`、`renderBrushes`、`renderTerrainConfig`、`updateModeUI`、`updateCurriculumUI`）
- **收益**：`main.ts` 减少 ~200 行，DOM 操作从算法代码中彻底隔离

### 批次 4：训练引擎独立化
- **改动文件**：新建 `training-engine.ts`、`main.ts`
- **内容**：提取 `TrainingEngine` 类，纯逻辑不碰 DOM。包含 `trainSupervised`、`trainUnsupervised`、`adjustEpsilon`、`evaluateDataset`
- **收益**：`main.ts` 减少 ~220 行，训练逻辑可测试性大幅提升

### 批次 5：快照管理器独立化
- **改动文件**：新建 `snapshot-manager.ts`、`main.ts`
- **内容**：提取 `SnapshotManager` 类，封装 `recordSnapshotStats`、`updateSnapshotSlider`、`applySnapshot`
- **收益**：`main.ts` 减少 ~50 行，快照重计算逻辑可独立优化

### 批次 6：课程控制器独立化
- **改动文件**：新建 `curriculum-controller.ts`、`main.ts`
- **内容**：提取 `CurriculumController` 类，维护阶段索引和运行状态。合并 `runCurriculumSupervised` 和 `runCurriculumUnsupervised` 的重复控制流，通过传入训练策略函数来区分模式
- **收益**：`main.ts` 减少 ~130 行，消除课程学习 80% 重复代码

### 批次 7：整理梯度接口重复 + 修复硬编码
- **改动文件**：`supervised.ts`、`unsupervised.ts`、可能新建 `gradients.ts`、`terrain.ts`
- **内容**：
  - 合并 `GradientBuffer`、`isValidGradientBuffer`、`getTotalGradientMagnitude` 到公共模块
  - 修复 `supervised.ts` 和 `unsupervised.ts` 中 `d < 2` 的硬编码，改为 `EMBED_DIM`
  - 提取 `terrain.ts` 中 `generateTerrainData` 和 `generateRandomTerrain` 的公共地形生成逻辑
- **收益**：消除接口重复和魔法数字

## 八、风险提示

1. **main.ts 中大量 `(window as any).xxx = xxx` 的全局函数暴露**：重构后要确保 HTML 的 `onclick` 仍能正确调用到新模块的方法。
2. **`state` 是全局单例**：新提取的类可能需要接收 `state` 引用，避免改成深拷贝导致状态不同步。
3. **` unsupervisedHistory` 和 `epsilon` 的窗口调整逻辑**：迁移到 `TrainingEngine` 时要保证状态引用一致。
4. **Canvas ResizeObserver 回调**：重构后 observer 仍要正确触发对应绘制函数。
