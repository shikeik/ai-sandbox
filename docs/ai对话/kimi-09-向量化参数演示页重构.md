### 已完成 ✅

1. **页面完全重写为 TypeScript**

   - HTML 只负责布局骨架

2. **地形系统升级为 5×3 结构**
   - x0 列：固定主角位置（天上=空气，地上=🦊狐狸，地面=平地）
   - x1~x4 列：可交互编辑的环境区域
   - 三层结构：天上 / 地上 / 地面

3. **元素定义（5种）**
   - 空气 ⬜ - 所有层可放
   - 平地 🟩 - 仅地面层
   - 史莱姆 🦠 - 仅地上层
   - 恶魔 👿 - 仅天上层
   - 金币 🪙 - 天上/地上层

4. **动作系统（4种）**
   - 走：前1地面=平地 + 前1天上≠恶魔 + 前1地上≠史莱姆
   - 跳：前1天上≠恶魔 + 前2地面=平地 + 前2天上≠恶魔 + 前2地上≠史莱姆
   - 远跳：前1/前2天上≠恶魔 + 前3地面=平地 + 前3天上≠恶魔 + 前3地上≠史莱姆
   - 走A：前1地面=平地（可击杀同层史莱姆，对天上恶魔无效）
   - 优先级：走 → 跳 → 远跳 → 走A

5. **监督学习训练器**
   - 规则 AI 自动生成 6000 条合法训练数据
   - 自动过滤无解/死局地形
   - Mini-batch SGD 训练
   - 实时显示准确率/损失

6. **合法性检查（分两组显示）**
   - ✅ 可哪些：列出所有可选动作
   - ❌ X哪些：列出每个不可选动作的具体原因

7. **统一绘制函数**
   - `drawTerrainGrid()`：环境编辑器、MLP 输入视图共用
   - 通过 DrawOptions 参数控制显示差异

8. **动画系统**
   - 预测后直接在编辑器 canvas 播放动画
   - 走/走A：直线移动
   - 跳：抛物线到前2
   - 远跳：更高抛物线到前3
   - 走A击杀史莱姆时动画中消失

## 二、关键技术决策记录

### 2.1 为什么从强化学习改为监督学习？

**问题**：原 fox-jump 是在线强化学习，AI 要自己探索试错，训练信号稀疏且延迟，导致：
- 需要几万局才能收敛
- 信用分配困难
- 震荡不稳定

**解决方案**：
- 用规则 AI 生成带标签的训练数据
- 变成标准的监督学习（图像分类问题）
- 6000 条样本 + 16 隐藏神经元，浏览器内几秒收敛到 95%+

### 2.2 走A的设计权衡

**需求**：面对 [天上恶魔, 地上史莱姆, 地面平地] 这种无解地形，需要有一个破局手段

**走A规则**：
- 条件：前1地面=平地（唯一条件）
- 效果：移动1格 + 击杀同层（地上层）史莱姆
- 限制：对天上恶魔无效

**为什么这样设计**：
- 走A是最后的兜底选项
- 不能杀恶魔迫使玩家优先选择"跳"或"远跳"来躲避
- 增加策略深度

### 2.3 为什么去掉"下蹲"？

下蹲原意是躲避天上恶魔，但：
1. 与"走"的地面条件重复
2. 没有独特的战术价值
3. 增加复杂度

用走A替代后，系统更简洁。

### 2.4 合法性检查为什么分两组？

**问题**：用户想验证生成规则是否正确

**方案**：
- 第一组：✅ 可哪些（如果合法）
- 第二组：❌ X哪些（如果不合法，列出具体原因）

**坑**：最初 `checkValidity()` 函数名与 HTML5 原生 API 冲突，导致点击无响应。后来改名为 `validateTerrain()` 解决。

## 三、踩过的坑

### 3.1 HTML5 API 命名冲突

**现象**：合法性检查按钮点击无反应
**原因**：`checkValidity()` 是 `HTMLButtonElement` 的原生方法
**解决**：改名为 `validateTerrain()`

### 3.2 跳/远跳的落点判定错误

**现象**：全空气地形也显示"可跳/可远跳"
**原因**：旧逻辑只检查起跳点，没检查落点地面是否为平地
**解决**：修正判定逻辑
- 跳：落点 = 前2格
- 远跳：落点 = 前3格

### 3.3 训练数据混入死局

**现象**：网络准确率卡在 60% 上不去
**原因**：`getLabel()` 对死局返回默认值 0（走），把无解地形标记为"走"
**解决**：
- 死局返回 -1
- 数据生成时过滤 `isValidTerrain() === false` 的样本

### 3.4 TypeScript 引号规范

**现象**：ESLint 报错 "Strings must use doublequote"
**解决**：运行 `npm run lint:fix` 自动修复

## 四、统一绘制函数的设计

### 4.1 接口定义

```typescript
interface DrawOptions {
  cellW: number
  cellH: number
  gapX: number
  gapY: number
  startX: number
  startY: number
  showHero?: boolean      // 是否显示狐狸
  heroCol?: number        // 狐狸列位置
  heroRow?: number        // 狐狸行位置
  hideSlimeAt?: number | null  // 哪列的史莱姆被击杀（动画用）
  dimNonInteractive?: boolean  // 是否淡化主角列（MLP视图用）
}
```

### 4.2 使用场景

| 场景 | 参数设置 |
|------|---------|
| 编辑器正常态 | showHero=true, heroCol=0, heroRow=1 |
| 动画播放 | showHero=false, hideSlimeAt=1（走A击杀时） |
| MLP视图 | dimNonInteractive=true, showHero=true |
