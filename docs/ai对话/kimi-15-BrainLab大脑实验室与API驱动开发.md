## 一、背景与动机

### 1.1 空虚感的来源

用户在完成两个感知机演示页面后产生"空虚感"——这些 AI 只是条件反射，缺乏真正的智能：

> "他们根本不算智能，只是和逻辑机一样的低级反应而已"

**核心问题**：
- 狐狸只会"往右走"这一个任务
- 输入→输出的条件反射，没有"为什么"
- 没有意图，没有计划，没有"在乎"

### 1.2 关键洞察

**智能的标志不是"能做多复杂的事"，而是"能把复杂的事分解成自己制定的子目标"**。

一只被训练来"往右走"的狐狸，无论多熟练，都只是条件反射。一只会自己决定"先拿血包再往右"的狐狸，才看起来像是在思考。

## 二、核心架构：大小脑分工

### 2.1 双系统架构

```
┌─────────────────────────────────────────┐
│              大脑（Cortex）              │
│  ┌─────────────────────────────────┐   │
│  │  子目标生成器（Goal Generator）   │   │
│  │  根据当前状态和需求，生成子目标队列 │   │
│  │  - 生存需求：HP低 → 找血包        │   │
│  │  - 效率需求：时间紧 → 直走        │   │
│  │  - 安全需求：敌人多 → 绕路/利用机关 │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │  世界模型（World Model）          │   │
│  │  "如果我执行动作X，会发生什么？"    │   │
│  │  - 预测下一帧状态                 │   │
│  │  - 预测奖励/危险                  │   │
│  │  - 评估延迟后果                   │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │  子目标调度器（Scheduler）        │   │
│  │  维护当前子目标队列               │   │
│  │  [拿血包] → [去终点]              │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
                          ↓ 当前子目标 + 视野 + 想象结果
┌─────────────────────────────────────────┐
│              小脑（Cerebellum）          │
│  ┌─────────────────────────────────┐   │
│  │  模式匹配网络                    │   │
│  │  "在这种地形下，如何执行子目标"    │   │
│  │  （这就是你现在的感知机做的事）     │   │
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │  动作执行器                      │   │
│  │  走/跳/攻击的具体时机控制         │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 2.2 根本区别

| 维度 | 小脑（Cerebellum） | 大脑（Cortex） |
|------|-------------------|---------------|
| **时间尺度** | 即时反应（毫秒级） | 延迟规划（秒级） |
| **目标** | 单一隐式目标 | 显式子目标队列 |
| **决策依据** | 当前输入模式 | 想象的未来轨迹 |
| **可解释性** | 权重黑盒 | 子目标可观察、可干预 |

### 2.3 核心机制

**状态延迟转移**：
- 踩平台 → 触发机关 → 尖刺坠落 → 击杀敌人 → 路通
- 小脑只能学习"直接关联"（"看到敌人→躲开"）
- 大脑能理解"先靠近敌人是为了消灭敌人"

**世界模型的价值**：
```
狐狸走到悬崖边：
  小脑说"跳！"（训练过的本能）
  大脑说"跳会死"（预见危险）
  → 于是否决了小脑的提议，选择绕路
```

## 三、API 驱动开发：一切操作通过 HTTP

### 3.1 设计理念

**所有功能都暴露为 HTTP API**，而非直接调用函数。这使得：

1. **可测试性**：用 curl 即可测试任何功能
2. **可观察性**：通过 API 获取日志、状态、中间结果
3. **解耦**：UI 层与逻辑层完全分离
4. **多客户端**：同一 API 可被 Web、CLI、其他服务调用

### 3.2 API 设计

```
GET/POST /api/brain-lab/state      # 获取当前状态
POST     /api/brain-lab/step       # AI思考并执行一步
POST     /api/brain-lab/move       # 手动执行动作
POST     /api/brain-lab/think      # 只思考不执行
POST     /api/brain-lab/reset      # 重置游戏
POST     /api/brain-lab/set-depth  # 设置想象深度
POST     /api/brain-lab/logs       # 获取日志
POST     /api/brain-lab/clear-logs # 清空日志
```

### 3.3 curl 测试示例

```bash
# 查看初始状态
curl -X POST http://localhost:4000/api/brain-lab/state

# AI思考并执行一步
curl -X POST http://localhost:4000/api/brain-lab/step

# 手动向右移动
curl -X POST http://localhost:4000/api/brain-lab/move \
  -H "Content-Type: application/json" \
  -d '{"action":"RIGHT"}'

# 设置想象深度为5
curl -X POST http://localhost:4000/api/brain-lab/set-depth \
  -H "Content-Type: application/json" \
  -d '{"depth":5}'

# 获取调试日志
curl -X POST http://localhost:4000/api/brain-lab/logs
```

### 3.4 日志系统

API 自动收集所有操作日志：

```typescript
class GameInstance {
	logs: Array<{time: string, tag: string, msg: string}> = []
	
	log(tag: string, msg: string) {
		const time = new Date().toLocaleTimeString()
		this.logs.push({ time, tag, msg })
		console.log(`[BRAIN-LAB] [${tag}] ${msg}`)
	}
}
```

通过 `/logs` 端点可以获取完整操作历史，用于调试和问题定位。

## 四、实现细节

### 4.1 左下坐标系

- `y=0` 在最下面（地面）
- `y` 值越大位置越高
- 渲染时从下到上遍历，但显示时顶部是天空

### 4.2 机关机制

```
按钮在 (4,2)
尖刺在 (4,3) —— 按钮正上方
敌人在 (4,1) —— 按钮正下方

当狐狸踩到按钮 (4,2)：
  → 触发机关
  → 尖刺落下
  → 杀死下方敌人 (4,1)
  → 路通
```

### 4.3 AI 决策流程

```
每一步：
1. 想象 LEFT:  预测位置 + 奖励
2. 想象 RIGHT: 预测位置 + 奖励
3. 想象 JUMP:  预测位置 + 奖励
4. 想象 WAIT:  预测位置 + 奖励
5. 选择奖励最高的动作执行
```

## 五、关键认知洞见

### 5.1 环境复杂度决定 AI 形态

| 环境复杂度 | 单目标足够？ | 例子 |
|-----------|------------|------|
| 直线往右走 | ✅ 足够 | 现在的狐狸 |
| 有岔路需要选择 | ❌ 需要子目标 | "先去血包还是直接去终点？" |
| 有先后顺序 | ❌ 需要子目标 | "必须先拿钥匙才能开门" |
| 有资源冲突 | ❌ 需要子目标 | "血包有限，多个 AI 竞争" |

### 5.2 为什么需要"想象"

**只有小脑的 AI**：
- 看到敌人 → 只会躲 → 永远到不了终点

**有大脑的 AI**：
- 想象"跳上去踩按钮"的后果
- 发现可以杀死敌人
- 选择跳跃 → 到达终点

**关键**：大脑能**在想象中承受反直觉的行为**，因为看到了**延迟的正面后果**。
