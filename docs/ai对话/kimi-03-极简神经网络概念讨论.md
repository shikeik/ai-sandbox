## 二、概念详解（对话整理）

### 1. 坑和平地的数值表示

**用户最初的直觉**：
- 坑 = 0（空的意思）
- 平地 = 1（有东西）

**后来的直觉**（更符合直觉）：
- 坑 = 1（看到1就觉得有问题）
- 平地 = 0（看到0没问题）

**结论**：两种都可以，选择坑=1，平地=0，因为"看到1就知道要处理了"更符合直觉。

```javascript
输入：
  前一格是坑？  1=是，0=不是
  前两格是坑？  1=是，0=不是
  前三格是坑？  1=是，0=不是
```

### 2. 权重（旋钮）的初始值

**用户的疑问**：为什么把第1个旋钮调那么大（0.5），后面都是0？

**澄清**：
- 初始值应该全部设为0（无偏向）
- 0.5只是举例，不是初始值
- 初始全0时：
  ```
  想跳 = 0×0 + 0×0 + 0×0 + 0 = 0
  跳跃概率 = sigmoid(0) = 0.5  // 一半一半，完全随机
  ```

**结论**：3个旋钮（w1,w2,w3）初始都是0，bias也是0。

### 3. Sigmoid激活函数

**公式**：`sigmoid(x) = 1 / (1 + e^(-x))`

**用户疑问**：
- 这是行业标准吗？
- 实际什么意思？
- 为什么用这么复杂的公式？

**解释**：

**是行业标准**，1938年就有的标准公式。

**实际意思**（不用数学符号）：
- e ≈ 2.718（和圆周率π一样的基础常数）
- 公式意思是：把任意数字压缩成0~1之间的概率

**举例**：
- x = 0：结果 = 0.5（中间）
- x = 1：结果 ≈ 0.73（大概率）
- x = -1：结果 ≈ 0.27（小概率）

**为什么要用这个？**
1. 能把任何数字（负无穷到正无穷）压缩成0~1
2. 平滑、连续（方便计算调整方向）
3. 行业标准，大家都用

**图形理解**：
```
输出1.0 |                    ****** 
      |               *****
      |          ****
      |      ****
      |   ***
输出0.5 |  **
      | *
      |*
      +-------------------------
      -5   0   5   输入
```

### 4. 偏置（bias）是什么

**用户的理解**：就是一个系数来调整偏向。

**详细解释**：

bias = 基础跳跃倾向（没有输入时的默认值）

**例子**：
```javascript
想跳 = 前一格×旋钮1 + 前两格×旋钮2 + 前三格×旋钮3 + bias;
```

- bias = 0：无偏向，50%跳跃
- bias = 2：爱冒险，没事也想跳
- bias = -2：保守，有坑才跳

**结论**：初始bias=0（无偏向）。

### 5. 为什么需要激活函数（线性 vs 非线性）

**用户背景**：看过函数逼近的教程，知道激活函数能让网络拟合曲线而不是直线。

**针对游戏的解释**：

**线性** = 直线，比如 `y = 2x + 3`
- 只能按比例响应
- 无法做"条件判断"

**问题**：
```
跳跃概率 = 旋钮 × 前一格
```
如果旋钮=0.5：
- 前一格=0（坑），概率=0（坚决不跳）
- 前一格=1（平地），概率=0.5（还跳？）

**问题**：平地时应该坚决不跳，但直线做不到"0或1"的切换。

**非线性（ReLU激活函数）的作用**：
```
如果输入 < 0 → 输出0（完全关闭）
如果输入 > 0 → 输出输入（正常通过）
```

这就像**开关**：
- 小于阈值 → 灯灭
- 大于阈值 → 灯亮

有了开关，就能表达复杂逻辑：
- "前一格是坑" 且 "前两格不是坑" → 跳
- "前两格是坑" → 不跳（即使前一格不是坑）

### 6. 单层 vs 多层网络（隐藏层）

**单层网络**（输入直接到输出）：
```
前一格 ──→ 跳不跳
前两格 ──→ 跳不跳
前三格 ──→ 跳不跳
```
- 每个人直接投票
- 不知道别人看到了什么

**多层网络**（加隐藏层）：
```
第一层（特征提取）：
  神经元A："前一格是坑吗？"
  神经元B："前两格是坑吗？"
  神经元C："前1格是坑但前2格不是？"

第二层（决策）：
  如果A激活 → 倾向跳
  如果B激活 → 倾向不跳（会跳进坑）
  如果C激活 → 必须跳（唯一生路）
```

**形象比喻**：
- 单层 = 直接投票，不知道别人想法
- 多层 = 先开会讨论，再决策

**当前游戏**：
- 现在只有单格坑，单层够用
- 将来有连续两格坑，可能需要多层（需要知道"前两格都是坑"这个组合信息）

### 7. 梯度下降

**用户状态**：还不知道什么意思。

**解释（不用数学）**：

**场景**：你在山上，要找最低点（最佳权重）。

**笨办法（随机搜索）**：
- 到处乱走，试试哪个位置最低
- 慢，可能永远找不到

**梯度下降（看坡度）**：
- 站在这里，看看哪个方向下坡最陡
- 往那个方向走一步
- 重复

**在神经网络中**：
- "山的高度" = 错误程度（死亡次数）
- "位置" = 旋钮的值
- "坡度" = 如果把旋钮调大一点，错误会增加还是减少？

**怎么算坡度？**（简化版）
- 死亡后，你知道刚才旋钮1的值是0.5
- 如果你把它改成0.6，下次会不会更好？
- 通过尝试微小变化，看结果怎么变，这就是梯度

### 8. 反向传播

**用户状态**：已经懂了。

**用户的理解**：
死亡或存活后，从输出往回调整权重。

**详细解释**：

**核心思想**：
- 死亡了 → 刚才选择的动作是错的 → 降低导致这个选择的权重
- 存活了 → 刚才选择的动作是对的 → 提高导致这个选择的权重

**"责任分摊"**：
```
输入层（观察员）："我看到坑了！"
隐藏层（分析师）："根据观察，危险！"
输出层（决策者）："我选择移动" → 死亡

复盘：谁的责任？
- 决策者（输出层）：主要责任
- 分析师（隐藏层）：传递了错误信号
- 观察员（输入层）：信息是对的，但权重影响不对
```

**反向传播就是**：从输出层往回，按责任大小调整每个权重。

## 三、完整决策流程（整理版）

```javascript
function decide(前一格, 前两格, 前三格) {
    // 3个旋钮，初始都是0
    let 旋钮1 = 0;  // 前一格的影响力
    let 旋钮2 = 0;  // 前两格的影响力
    let 旋钮3 = 0;  // 前三格的影响力
    let bias  = 0;  // 基础跳跃倾向
    
    // 计算"想跳的程度"
    let 想跳 = 前一格×旋钮1 + 前两格×旋钮2 + 前三格×旋钮3 + bias;
    
    // 压缩成概率（Sigmoid）
    let 跳跃概率 = sigmoid(想跳);  // 0~1之间
    
    // 抛硬币决策
    if (Math.random() < 跳跃概率) {
        return '跳跃';
    } else {
        return '移动';
    }
}

// Sigmoid（压缩函数）
function sigmoid(x) {
    return 1 / (1 + Math.exp(-x));
}
```

**训练后预期的旋钮值**：
- 旋钮1 = 正数（比如0.8）：看到前一格坑→想跳
- 旋钮2 = 负数（比如-0.8）：看到前两格坑→不想跳（跳了会死）
- 旋钮3 ≈ 0：暂时没用到（预留）
