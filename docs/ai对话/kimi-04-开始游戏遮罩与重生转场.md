## 五、关键设计决策记录

### 决策1：状态分离架构
```javascript
// 游戏生命周期状态（控制能否操作）
game.gameStatus = READY | RUNNING | TRANSITIONING | FINISHED

// 人物动作状态（控制动画表现）
player.action = IDLE | MOVING | JUMPING
```

**理由**：避免语义混淆，RUNNING 表示"游戏进行中"，IDLE 表示"人物待机"。

### 决策2：连续操作机制
**不是**：动画中按键 → 预存 → 动画完自动执行  
**而是**：动画中按键 → 立即打断 → 新动作立即生效  

**理由**：速通核心体验，玩家按键频率直接决定角色移动速度。

### 决策3：TIME/BEST 同时显示
**从**：动态切换（RUNNING=TIME，其他=BEST）  
**改为**：始终同时显示  

**理由**：速通玩家需要始终关注当前进度和历史最佳。

## 六、遇到的问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 一次操作后无法继续 | `action !== IDLE` 检查阻止了连续操作 | 移除检查 |
| TIME 自动变 BEST | `notifyVisualComplete` 将状态设为 IDLE | 改为 RUNNING |
| 转场亮起后才重置 | `onGenerationChange` 调用时机太晚 | 移到 `_executeRespawn` |
| BEST 显示为 0 | localStorage 存储了无效值 | 添加有效性检查 |
