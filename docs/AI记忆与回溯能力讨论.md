# AI 记忆与回溯能力讨论

> 我们现在的 AI 有个根本缺陷：它只有"反应"，没有"记忆"。这篇文档记录这个问题的本质，以及可能的解决路径。

---

## 一、问题本质：为什么 AI 不会"回头"

设想一个场景：

```
[起点] → [平地] → [岔路：上/下] → AI 选择了"下"
                                  ↓
                          [陷阱：死胡同]
```

如果 AI 走到死胡同发现前面是坑，它应该做什么？

**人类会做的**："啊这条路不通，我回到刚才的岔路口，试试另一条路。"

**我们现在的 AI 会做的**："当前状态 = 面前是坑。最佳动作 = 跳。跳不过去？死。"

因为它看不到"身后还有一条岔路"，也记不住"自己曾经做过选择"。

### 根因

当前 `fox-jump` 和 `terrain-lab` 的神经网络都是**前馈网络（Feedforward）**：

```
输入 = 当前一帧的感知（4 格地形 或 5×3 网格）
     ↓
MLP / 感知机
     ↓
输出 = 当前帧的动作
```

这是一个**单帧输入 → 单帧动作**的映射。网络没有"隐藏状态"来携带历史信息，也没有一个"外部记忆库"来存储过去的状态。

---

## 二、记忆的层次与实现方案

从前端到后端、从简单到复杂，给 AI 加记忆有很多条路。下面按**实现难度**和**效果增益**排列。

### 方案 1：历史窗口（最简单，马上就能做）

**原理**：不再只把"当前帧"作为输入，而是把"最近 N 帧"拼接成一个更大的向量。

```
输入 = [ frame(t-2), frame(t-1), frame(t) ]
```

例如 `terrain-lab` 中，单帧是 90 维。取最近 3 帧，输入就变成 270 维。

**优点**：
- 无需改动网络结构，只需要把输入层扩大
- AI 能感知到"趋势"（比如敌人在靠近）
- 实现成本极低

**缺点**：
- 无法记住很久之前的事（N 不能太大，否则输入维度爆炸）
- 无法主动"回溯"或"规划"

**适用性**：★★★☆☆ 适合我们的项目，可以立刻尝试。

---

### 方案 2：循环神经网络 RNN / LSTM / GRU（中等难度）

**原理**：在网络中加入一个"隐藏状态"（hidden state），它像一条链条一样把每一帧的信息传递下去。

```
输入 = 当前帧
隐藏状态 = f(上一帧隐藏状态, 当前输入)
输出 = 基于隐藏状态的动作
```

LSTM（长短期记忆网络）是 RNN 的改进版，专门解决"信息传太远会消失"的问题。

**优点**：
- 理论上可以记住很久之前的经历
- 网络本身就能学到"什么时候该记住，什么时候该忘记"
- 是早期 AlphaStar、OpenAI Five 等游戏的基线方案

**缺点**：
- 需要手写 RNN/LSTM 的前向传播（如果是纯前端 TS 的话）
- 训练更慢，梯度更容易出问题
- 仍然是在做"反应"，而不是"规划"——它有记忆，但不一定会"主动回溯"

**适用性**：★★★★☆ 对平台跳跃游戏很有价值，但需要一定开发量。

---

### 方案 3：Transformer / 自注意力机制（较高难度）

**原理**：把最近 N 帧当作一个"序列"，网络通过"自注意力"（Self-Attention）自动决定"哪一帧的历史信息对当前决策最重要"。

```
序列 = [frame(t-10), frame(t-9), ..., frame(t)]
输出 = 对每个历史帧加权求和后的决策
```

ChatGPT 和 AlphaFold 的核心都是这个机制。

**优点**：
- 可以同时关注远处的"岔路口"和近处的"陷阱"
- 没有 RNN 那种信息逐层衰减的问题
- AlphaStar 的最终版本用的就是 Transformer

**缺点**：
- 计算量比 LSTM 更大
- 在前端纯手写 Transformer 非常困难（矩阵乘法、多头注意力、位置编码）
- 需要更多训练数据才能稳定

**适用性**：★★☆☆☆ 对我们来说太复杂了，除非引入 TensorFlow.js 等库，否则不建议手写。

---

### 方案 4：显式外部记忆 + 经验回放（算法层面）

**原理**：网络本身不变，但在训练时维护一个"记忆库"（Replay Buffer），存储过去遇到的所有 `(状态, 动作, 奖励, 新状态)` 四元组。训练时随机抽取记忆进行回放。

```
记忆库 = [
  {s: 岔路口, a: 选择下路, r: 0, s': 平地},
  {s: 死胡同, a: 跳, r: -500, s': 死亡},
  ...
]
```

这是 DQN（Deep Q-Network）的核心技术。

**优点**：
- 让网络学会"长期后果"（比如因为死胡同的 -500 惩罚，网络会重新评估"选择下路"这个动作的价值）
- 不需要改动网络结构
- 训练更稳定，数据利用率更高

**缺点**：
- 解决的是"学习"问题，不是"推理时的记忆"问题
- AI 在运行（推理）时仍然只看当前帧，不会"主动想起来"要去岔路口

**适用性**：★★★★★ 非常适合我们，是强化学习的必经之路。

---

### 方案 5：规划 + 神经网络（最高级，也最有效）

**原理**：AI 不是单靠神经网络做决策，而是分为两层：
- **下层（神经网络）**：负责局部感知和快速反应（看到坑就跳）
- **上层（规划器）**：负责全局搜索和回溯（A*、蒙特卡洛树搜索 MCTS）

```
地图记忆 = 已探索过的所有格子及其连接关系
规划器 = "从当前位置出发，搜索到终点的最短安全路径"
神经网络 = "执行规划器给出的下一个动作"
```

**优点**：
- 真正解决了"回溯到岔路口"的问题
- AlphaGo、AlphaZero 就是"神经网络 + MCTS"的组合
- 规划器是可解释的，你可以看到 AI"在想什么"

**缺点**：
- 实现 MCTS 或 A* 需要完整的"世界模型"（AI 要能模拟"如果我做这个动作，世界会变成什么样"）
- 在前端实时运行搜索树，计算量可能有点大

**适用性**：★★★★☆ 非常适合网格化马里奥。网格环境本身就很适合 A* 搜索。

---

## 三、"回到岔路口"这个问题的真正解法

如果我们想要 AI 在死胡同里能**主动回头**，单靠神经网络（无论有没有 LSTM）是不够的。

因为这涉及到一个更高级的认知能力：**规划（Planning）**。

### 为什么神经网络本身不会规划

神经网络做的是"模式匹配"：
- 输入 = 当前看到的画面
- 输出 = 直觉上最正确的动作

它从来没有"在脑子里推演"过"如果我回头会怎样"。

### 怎么才能让 AI 回头

最务实的做法，是**把"记忆"和"规划"拆成两个系统**：

1. **记忆系统（Memory Map）**
   - 维护一张 AI 已经探索过的地图
   - 记录每个格子走过没有、是否安全、是否是死胡同

2. **规划系统（Planner）**
   - 当 AI 发现当前位置没有安全出路时，搜索记忆地图
   - 找到"离当前位置最近的、还有未探索分支的安全节点"
   - 生成一条返回路径

3. **执行系统（神经网络 / 控制器）**
   - 沿着规划器给出的路径，一步一步执行动作（走、跳、躲避敌人）

这其实就是**人脑的工作方式**：
- 小脑（神经网络）= 平衡、闪避、跳跃时机
- 前额叶皮层（规划器）= "这条路不通，我得回去试试另一条"

## 四、对我们项目的渐进建议

结合我们纯前端、手写 TS 的定位，建议按以下顺序攻克：

### 第一步：加入历史窗口（立刻可做）
- 把 `terrain-lab` 的输入从单帧 90 维扩展为"最近 3 帧共 270 维"
- 让 AI 学会"敌人正在靠近，我要提前跳"

### 第二步：引入经验回放（强化学习必经之路）
- 把训练从监督学习改成 DQN 风格的 Q-Learning
- 加入 Replay Buffer，让 AI 学会"某些动作的长期后果很糟糕"

### 第三步：加入显式地图记忆（网格化马里奥的核心）
- 在网格化马里奥中，维护一张 `visited[][]` 和 `safe[][]` 的二维表
- 当 AI 被困住时，规划器搜索最近的未探索安全节点

### 第四步（远期）：轻量级规划器
- 在网格世界中实现一个简单的 A* 搜索
- 神经网络负责"局部反应"（躲子弹、跳坑），A* 负责"全局路径"（去哪里）

---

## 五、结论

- **当前 AI 没有记忆**，是因为前馈网络本质上就是"单帧映射"。
- **真正的"回溯到岔路口"不是记忆问题，而是规划问题**。需要网络之外的一个规划器。
- **给 AI 加记忆有清晰的渐进路径**：历史窗口 → 经验回放 → 外部地图记忆 → 规划器。
- 对我们的项目来说，**"经验回放 + 显式地图记忆"是最值得投入的方向**，它能让 AI 从一个"只会条件反射的动物"进化成"有目标、能探索、会回头的智能体"。
