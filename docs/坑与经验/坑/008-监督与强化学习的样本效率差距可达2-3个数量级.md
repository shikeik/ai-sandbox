# 监督与强化学习的样本效率差距可达 2~3 个数量级

## 现象
同样的简单决策任务，监督学习 6000 条样本几秒收敛，纯强化学习随机探索几小时都没明显进步。

## 根因
| 学习方式 | 信号强度 | 原因 |
|---|---|---|
| 监督学习 | 极强 | 每一步都有"正确答案"直接修正网络 |
| 强化学习 | 极弱 | 只有最终结果（死亡/存活），且存在信用分配问题 |

三座大山：
1. **信用分配问题**（Credit Assignment）：连续 10 步后死亡，网络不知道哪一步该负责
2. **探索效率极低**：随机策略在复杂地形下存活率可能低于 0.1%
3. **稀疏奖励**：大多数尝试没有任何有效信号

## 量化对比
| 方式 | 达到可玩水平所需样本 |
|---|---|
| 监督学习 | ~6,000 条 |
| 强化学习（有奖励塑形+课程学习+经验回放） | ~10万 ~ 100万步 |
| 强化学习（稀疏奖励，无课程） | 可能永远学不会 |

## 解决
如果项目定位是**浏览器内实时教学演示**，不要尝试纯摸索：
1. 先做监督学习或模仿学习给网络一个正确先验
2. 再引入奖励塑形（每步都给中间奖励）
3. 使用课程学习从简单到复杂渐进解锁
4. 加入经验回放（Experience Replay）
