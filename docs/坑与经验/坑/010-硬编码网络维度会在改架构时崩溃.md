# 硬编码网络维度会在改架构时崩溃

## 现象
把 `hidden_dim` 从 16 改成 32 后，训练正常，但测试文件运行时数组越界、梯度维度不匹配、 silently 输出错误结果。

## 根因
测试文件或辅助函数里写了 `Array(16)`、`Array(2)` 等硬编码，没有跟随主代码的常量变。

## 解决
所有与网络结构相关的数值必须提取为常量：
```typescript
const HIDDEN_DIM = 32
const EMBED_DIM = 2
const NUM_ELEMENTS = 6

// 不要这样
const buf = Array(16).fill(0)

// 要这样
const buf = Array(HIDDEN_DIM).fill(0)
```

## 建议
修改网络超参数时，全局搜索项目里的所有数字字面量，确认没有遗漏的硬编码。
