# 数据合法性检查与预测必须共用同一数据源

## 现象
系统显示"预测合法"，但验收时标为"❌ 错误"；或者生成数据时过滤掉了死局，但训练标签里仍包含死局默认值。

## 根因
`predict()`、`getLabel()`、`validateTerrain()` 分别实现了各自的判定逻辑，导致同一地形在不同地方结论不一致。

## 解决
**所有入口必须调用同一套判定函数**。例如：
```javascript
// 唯一真值源
const checks = getActionChecks(terrain, heroCol)
const isValid = isActionValidByChecks(checks, action)
const label = getLabel(terrain) // 内部也要基于 checks 计算

// 预测、验收、数据生成全部使用 isActionValidByChecks
```

## 经验
任何涉及"规则判断"的代码，不要写多个版本的判断逻辑。哪怕只是复制粘贴，后期一改就不同步。应该提取一个纯函数作为**唯一真值源**。
