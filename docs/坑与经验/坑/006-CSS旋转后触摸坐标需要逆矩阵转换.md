# CSS 旋转后触摸坐标需要逆矩阵转换

## 现象
对容器应用 `transform: rotate(-90deg)` 后，手指在屏幕上向上滑动，画面里的角色却往左走。

## 根因
CSS 旋转改变了视觉坐标系，但 `touchstart`/`touchmove` 返回的仍然是屏幕原始坐标。如果不做逆变换，逻辑坐标和视觉坐标完全错位。

## 解决
在 `rotate(-90deg)` 的情况下，逆变换公式：
```javascript
// 假设容器中心为旋转原点
const logicX = virtualWidth - (screenY / scale)
const logicY = screenX / scale
```

**通用做法**：先减去旋转中心，应用逆旋转矩阵，再加回中心。

## 建议
如果项目需要频繁处理旋转后的交互，建议把坐标转换封装成 `toVirtualCoordinates(screenX, screenY, rotation)` 工具函数。
