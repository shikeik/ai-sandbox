# 状态分离：生命周期状态与动画状态应分两层

## 场景
游戏角色有"待机、移动中、跳跃中"等动画，同时游戏本身有"准备中、进行中、转场中、已结束"等生命周期。

## 反模式
把两者混为一层：
```javascript
const state = READY | RUNNING | IDLE | MOVING | JUMPING | DEAD
```
这会导致语义混乱：RUNNING 到底是指"游戏进行中"还是"奔跑动画"？

## 正例
```javascript
// 游戏生命周期（控制能否操作）
const GAME_STATUS = { READY: 0, RUNNING: 1, TRANSITIONING: 2, FINISHED: 3 }

// 人物动画状态（控制表现）
const PLAYER_ACTION = { IDLE: 0, MOVING: 1, JUMPING: 2 }
```

## 好处
- 转场时可以把 `GAME_STATUS` 设为 `TRANSITIONING`（禁止输入），但 `PLAYER_ACTION` 仍保持 `JUMPING`（动画继续播完）
- 死亡时 `GAME_STATUS = FINISHED`，但转场动画独立管理
- 连续操作机制只需要检查 `GAME_STATUS === RUNNING`，不需要关心当前动画
