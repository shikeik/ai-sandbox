# 动态铺满布局：取屏幕短边作为设计基准

## 场景
移动端 H5 游戏需要适配各种尺寸：手机竖屏、手机横屏、平板、PC 桌面窗口。

## 方案
放弃固定分辨率（如 1080×1920），改为：
```javascript
const virtualWidth = Math.min(physWidth, physHeight)
const virtualHeight = virtualWidth * (16 / 9) // 或你的设计比例
```

这样在任何设备上都能**取短边做宽，长边做高**，自动铺满不留黑边。

## 对比
| 方案 | 优点 | 缺点 |
|---|---|---|
| 固定 1080×1920 | 设计简单 | 在手机上 CSS 像素只有 384px，显得极小 |
| 动态取短边 | 铺满、无黑边 | 内部元素大小需要相对单位或 clamp 控制 |

## 配套经验
- 按钮尺寸用 `min(100px, 20vw)` 而非固定 px
- 文字用 `clamp(12px, 3vw, 18px)`
- 卡片高度用 flex 比例（如 `flex: 4 4 2`）而非百分比
