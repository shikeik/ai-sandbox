# 浏览器 API 不支持时要检测再降级

## 场景
使用了 `screen.orientation.lock()`、`requestFullscreen()`、`navigator.share()` 等较新的浏览器 API。

## 核心经验
**不要假设 API 存在，也不要假设报错后用户能看懂。**

## 检测模式
```javascript
const supportsOrientationLock = !!screen.orientation?.lock
const isEdge = /Edg/.test(navigator.userAgent)

if (isEdge || !supportsOrientationLock) {
  // 显示友好提示，而不是让 API 报错
  showToast("请使用 Chrome 浏览器以获得最佳体验")
}
```

## 为什么重要
- iOS Safari 不支持方向锁定
- Via/夸克等国产浏览器未实现 Screen Orientation API
- Edge 有刘海黑边问题
- 用户的浏览器环境千变万化，提前检测比事后救火成本低得多
