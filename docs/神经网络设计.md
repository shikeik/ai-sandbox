## 1. 已确定的设计决策

### 1.1 输入层

**视野范围**：3格
- 前一格（当前位置+1）
- 前两格（当前位置+2）
- 前三格（当前位置+3）

**数值表示**：
- 坑 = 1
- 平地 = 0

```javascript
输入格式：[grid1, grid2, grid3]
// grid1: 前一格是坑? (0或1)
// grid2: 前两格是坑? (0或1)
// grid3: 前三格是坑? (0或1)
```

**理由**：
- 当前游戏只有单格坑，但预留视野为未来"连续两格坑"做准备
- 3格视野在将来加入"跳两格"能力时仍然够用

---

### 1.2 输出层

**动作数量**：2个
- 动作0：移动（右移1格）
- 动作1：跳跃（右移2格）

**输出格式**：概率分布
```javascript
输出：[prob_move, prob_jump]
// prob_move: 选择移动的概率 (0~1)
// prob_jump: 选择跳跃的概率 (0~1)
// 且 prob_move + prob_jump = 1
```

---

### 1.3 激活函数

**选择**：Sigmoid

**公式**：
```
sigmoid(x) = 1 / (1 + e^(-x))
```

**作用**：将任意数值压缩到 0~1 范围，表示概率

**特性**：
- 输入 0 → 输出 0.5
- 输入很大正数 → 输出接近 1
- 输入很大负数 → 输出接近 0

---

### 1.4 学习率与权重更新规则

**基础学习率**：α = 0.2

**权重初始化**：全0
- 初始输出概率 50/50
- 无偏向，完全依赖训练

**更新规则（非对称）**：
```javascript
// 存活/胜利时（正确）
选中的动作权重 += 0.2;

// 死亡时（错误）
选中的动作权重 -= 0.2;        // 惩罚选中的
其他每个动作权重 += 0.1;      // 奖励是惩罚的一半

// 示例（两个动作）：
// 选了移动，死亡
移动权重 -= 0.2;   // -0.2
跳跃权重 += 0.1;   // +0.1（0.2的一半）
```

**理由**：
- 基础值0.2比0.1学得快，适合确定性环境
- 错误时惩罚(0.2) > 其他奖励(0.1)，体现"确实错了" vs "可能只是对的"
- 其他动作均分奖励（多个动作时各+0.1）

---

### 1.5 偏置（Bias）

**决定**：不使用偏置

**理由**：
- 偏置是"人工设定偏好"（比如天生爱冒险）
- 希望AI完全从0开始学习，不预设任何偏向
- 简化网络结构

## 2. 待确定的决策

### 2.1 网络结构

**选项A：单层（感知机）**
```
输入(3) ──→ 输出(2)
```
- 简单，只有 3×2=6 个权重
- 每个输入直接连接到每个输出

**选项B：带隐藏层**
```
输入(3) ──→ 隐藏层(4) ──→ 输出(2)
```
- 输入先经过中间层处理，再输出
- 参数更多：3×4 + 4×2 = 20个权重
- 能学习组合特征（如"前两格都是坑"）

**状态**：待用户理解隐藏层概念后确定

---

### 2.2 探索策略

**选项A：纯贪心**
- 总是选择概率高的动作
- 简单，但可能陷入局部最优

**选项B：ε-贪心（Epsilon-Greedy）**
- 80%~90%按概率选（利用已知）
- 10%~20%随机选（探索新可能）
- 平衡探索与利用

**状态**：待解释后确定

## 3. 关键概念详解（供理解）

### 3.1 隐藏层是什么？

**类比：公司决策流程**

**单层（没有隐藏层）**：
```
员工A（看到坑）──┐
员工B（看到平地）─┼──→ 老板直接听汇报做决定
员工C（看到旗子）──┘
```
- 老板同时听3个人汇报
- 每个人直接投票
- **问题**：无法讨论"如果A和B同时发生怎么办"

**带隐藏层（有中间管理层）**：
```
员工A ──→ 经理甲（分析"危险度"）──┐
员工B ──→ 经理乙（分析"可行性"）─┼──→ 老板做决策
员工C ──→ 经理丙（分析"目标方向"）──┘
```
- 经理先加工信息（提取特征）
- 老板基于经理的总结做决策
- **优势**：能处理组合条件

**在你的游戏中**：
- **单层**：看到前一格坑→跳（直接映射）
- **隐藏层**：
  - 隐藏神经元A："前一格是坑"（危险信号）
  - 隐藏神经元B："前两格是坑"（跳跃不可行信号）
  - 输出层：综合A和B，决定跳不跳

**什么时候需要隐藏层？**
- 现在（单格坑）：不需要，单层够用
- 未来（连续两格坑）：需要，因为要判断"前两格都是坑→不能跳"

---

### 3.2 探索策略是什么？

**问题**：AI学会了"看到坑就跳"，但万一还有更好的策略呢？

**纯贪心（不探索）**：
- AI永远按当前最优选择
- 如果当前最优是错的，永远改不了
- 例子：AI认为"看到坑有90%概率跳"，永远不尝试"不跳"看结果

**ε-贪心（带探索）**：
- 90%按最优选（利用已有知识）
- 10%随机选（尝试新可能）
- 例子：偶尔尝试"不跳"，如果死了，确认"跳是对的"；如果活了，发现新策略

**在你的游戏中**：
- 环境是确定性的（坑=必死）
- 理论上不需要探索，因为试错一次就知道对错
- **建议**：先用纯贪心，如果学习停滞再加入探索
