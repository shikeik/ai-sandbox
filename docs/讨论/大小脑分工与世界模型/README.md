### 1. 大小脑分工与世界模型设计讨论.md
**阶段**：架构设想

从"为什么现在的AI只是条件反射"的空虚感出发，提出双系统架构：
- **小脑**：感知-反射网络（快速、短视）
- **大脑**：世界模型（想象引擎、可以离线推演）

核心洞察：狐狸走到悬崖边，小脑说"跳"（训练过的本能），大脑说"跳会死"（预见危险）——这就是真正的认知分离。

### 3. 新环境设计方案讨论.md
**阶段**：环境设计

讨论为什么当前环境"太简单不需要大脑"：
- 当前：只能往右走、即死机制、5x3视野 → 小脑足够
- 未来：血量系统、血包、11x5居中视野、左右自由移动、上下平台 → 需要大脑

核心机制：
- 血量低时需要在"拿血包绕路"和"硬闯"之间选择
- 跳下去上不来 → 需要预判后果
- 多条路径 → 需要比较未来收益

推荐方案：升级 grid-world 系统（而非改造fox-jump），新建"大脑挑战"页面。

### 4. 阶段三：核心机制设计与认知架构细化.md
**阶段**：核心机制设计

细化三大强制需要大脑的机制：

| 机制 | 核心特征 | 典型例子 |
|------|---------|---------|
| **状态延迟转移** | 动作效果多步后显现 | 尖刺陷阱：踩平台→尖刺掉→敌人死 |
| **子目标分解** | 大脑自主发布任务队列 | [拿血包]→[杀敌人]→[去终点] |
| **环境互动** | 工具使用+因果推理 | 钥匙开门、尖刺作为武器 |

**尖刺机制的精妙之处**：
- 创造"虚假危险"（需要主动靠近敌人）
- 强制因果推理（踩平台→机关→击杀→通路）
- 自然引出"顿悟学习"（试错→理解→主动应用）

## 核心概念速查

### 三层级认知架构
```
大脑（Cortex）
├── 子目标生成器（生存/效率/安全需求）
├── 世界模型（想象引擎）
└── 子目标调度器（动态维护队列）

小脑（Cerebellum）
└── 模式匹配网络（执行当前子目标）
```

### 小脑 vs 大脑
| | 小脑 | 大脑 |
|--|------|------|
| 时间尺度 | 毫秒级即时反应 | 秒级延迟规划 |
| 目标 | 单一隐式 | 显式子目标队列 |
| 决策依据 | 当前输入模式 | 想象的未来轨迹 |

### 学习四阶段
1. **无知**：躲开敌人（本能）
2. **偶然**：踩到平台发现机关（探索）
3. **顿悟**：主动规划利用机关（大脑主导）
4. **泛化**：识别"平台+尖刺=武器"模式
