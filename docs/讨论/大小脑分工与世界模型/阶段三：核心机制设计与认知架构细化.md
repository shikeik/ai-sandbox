# 阶段三：核心机制设计与认知架构细化

> 时间：2026-04-06
> 背景：在明确大小脑分工和世界模型方向后，进一步细化核心游戏机制和认知架构设计

---

## 一、从"空虚"到"顿悟"的认知演进

### 最初的空虚感
> 我感觉好空虚，两个页面整了两个小感知机ai，但是突然感觉也没什么意思，他们根本不算智能，只是和逻辑机一样的低级反应而已

**问题根源**：
- 狐狸只会"往右走"这一个任务
- 输入→输出的条件反射，没有"为什么"
- 没有意图，没有计划，没有"在乎"

### 关键洞察：复杂环境催生大脑

**用户的核心洞见**：
> 现在的AI是我们给它指定的有且只有一个任务，那就是往右走...但现在他需要自己给自己发布任务，然后通过这些任务逐步达成最终目标

**结论**：
- **小脑**：完成单一任务（往右走）
- **大脑**：发布多元化的子目标，指挥小脑逐个完成

## 二、认知架构：三层级系统

### 架构总览

```
┌─────────────────────────────────────────────────────────┐
│                    大脑（Cortex）                        │
│  ┌─────────────────────────────────────────────────┐   │
│  │  子目标生成器（Goal Generator）                   │   │
│  │  根据当前状态和需求，生成子目标优先级队列          │   │
│  │  - 生存需求：HP低 → 找血包                       │   │
│  │  - 效率需求：时间紧 → 直走                       │   │
│  │  - 安全需求：敌人多 → 绕路/利用机关              │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │  世界模型（World Model）                          │   │
│  │  "如果我执行动作X，会发生什么？"                   │   │
│  │  - 预测下一帧状态                                │   │
│  │  - 预测奖励/危险                                │   │
│  │  - 评估延迟后果                                  │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │  子目标调度器（Scheduler）                        │   │
│  │  维护当前子目标队列                              │   │
│  │  [拿血包] → [去终点] 或 [激活机关] → [通过区域]   │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                          ↓ 当前子目标 + 视野 + 想象结果
┌─────────────────────────────────────────────────────────┐
│                    小脑（Cerebellum）                    │
│  ┌─────────────────────────────────────────────────┐   │
│  │  模式匹配网络（现有感知机）                        │   │
│  │  "在这种地形下，如何执行子目标最安全的走/跳/攻击"  │   │
│  └─────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────┐   │
│  │  动作执行器                                       │   │
│  │  具体的走/跳/攻击动作控制                        │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 大脑与小脑的根本区别

| 维度 | 小脑（Cerebellum） | 大脑（Cortex） |
|------|-------------------|---------------|
| **时间尺度** | 即时反应（毫秒级） | 延迟规划（秒级） |
| **目标** | 单一隐式目标 | 显式子目标队列 |
| **学习方式** | 监督学习（成功/失败） | 自监督+想象推演 |
| **可解释性** | 权重黑盒 | 子目标可观察、可干预 |
| **决策依据** | 当前输入模式 | 想象的未来轨迹 |

## 三、核心游戏机制：强制需要大脑的三大要素

### 要素1：状态延迟转移（Delayed State Transitions）

**定义**：当前动作的效果不是立即显现，而是在多步后才发生

**典型例子**：尖刺陷阱机制

```
初始状态：
┌─────────────────────────┐
│  🦊      🟩            │  上层（平台）
│         🔺             │  尖刺倒挂在平台下
├─────────────────────────┤
│         👿      🏁     │  下层（地面）
│         [挡住路了]      │
└─────────────────────────┘

因果链（延迟5-10步）：
踩平台 → 触发机关 → 尖刺坠落 → 击杀敌人 → 路通 → 前进
```

**为什么小脑做不了**：
- 小脑只能学习**直接关联**（"看到敌人→躲开"）
- 无法理解**"先靠近敌人是为了消灭敌人"**这种反直觉策略
- 无法保持**延迟后果**在记忆中的激活

**大脑如何做到**：
- 世界模型预测："踩平台 → 尖刺掉 → 敌人死"
- 在想象中承受反直觉行为（"靠近敌人"）
- 评估长期收益 > 短期风险

---

### 要素2：目标的任务分解（Hierarchical Goal Decomposition）

**定义**：大脑将最终目标分解为可执行的子目标，动态调度

**典型场景**：血量管理 + 路径规划

```
当前状态：
- HP: 30%（碰敌人-30%，会死）
- 左边：血包（+30HP）
- 右边：敌人挡路 → 终点

大脑决策过程：
┌─────────────────────────────────────────┐
│ 主目标：到达终点                        │
├─────────────────────────────────────────┤
│ 直接路径评估：                          │
│ - 右边有敌人                            │
│ - 直接冲突：HP 30% - 30% = 0% → 死亡    │
│ - 结论：不可行                          │
├─────────────────────────────────────────┤
│ 子目标生成：                            │
│ 优先级1: [获取血包]                     │
│   - 路径：左移 → 左移 → 吃血包          │
│   - 风险：可能需要经过尖刺区域          │
│   - 收益：HP 30% → 60%                  │
│                                         │
│ 优先级2: [前往终点]                     │
│   - 前提：完成优先级1                   │
│   - 路径：返回 → 右移 → 击败/绕过敌人   │
│   - 现在可行：HP 60% - 30% = 30% > 0    │
├─────────────────────────────────────────┤
│ 调度决策：                              │
│ 当前激活子目标 = "获取血包"             │
│ 执行中...                               │
│                                         │
│ 事件触发：血包被其他AI抢走              │
│ 重新规划：                              │
│ 当前激活子目标 = "寻找替代路径"         │
│ 备选：硬闯（低概率成功）或 找其他血包    │
└─────────────────────────────────────────┘
```

**可视化设计**：
```
狐狸头顶显示：
┌────────────────────────┐
│ 当前目标：获取血包 (+30HP) │
│ 队列：[拿血包] → [去终点] │
└────────────────────────┘
```

### 要素3：环境互动机制（钥匙与尖刺）

#### 3.1 尖刺陷阱（因果推理）

**机制**：
- 尖刺倒挂在平台下方
- 踩平台 → 尖刺坠落 → 击杀下方敌人
- 敌人挡住必经之路，必须击杀

**认知要求**：
```
工具使用认知链：
1. 对象永久性：记住"那里有尖刺"（即使不在视野）
2. 功能可供性：平台不只是"可以站"，还是"可以触发机关"
3. 因果推理：A导致B，B导致C，所以A导致C
4. 反直觉执行：主动靠近敌人（看似危险）
```

**扩展变体**：

| 变体 | 复杂度 | 描述 |
|------|--------|------|
| 顺序触发 | ⭐⭐⭐ | 两个平台按特定顺序踩，才能清除两个敌人 |
| 资源选择 | ⭐⭐⭐ | 只有一个尖刺，两个敌人，需要选择杀哪个 |
| 自我伤害 | ⭐⭐⭐⭐ | 需要右移（远离目标）踩平台，再返回 |

#### 3.2 钥匙与门（长程依赖）

**机制**：
- 某些门需要特定颜色的钥匙
- 钥匙在地图另一端
- 必须先拿钥匙，再返回开门

**认知要求**：
```
子目标队列维护：
1. [寻找红钥匙] → 
2. [返回红门] → 
3. [通过红门] → 
4. [前往终点]

需要能力：
- 长程记忆：钥匙位置（探索后发现）
- 路径回溯：从终点反推到钥匙
- 子目标坚持：拿到钥匙前不被其他事物干扰
```

**钥匙-门的深度扩展**：

```
多层嵌套：
┌─────────────────────────────────────┐
│ 终点                                │
│  ↓ 需要蓝钥匙                        │
│ 蓝门                                │
│  ↓ 需要击败守卫（或绕路）             │
│ 守卫区                              │
│  ↓ 需要红钥匙才能通过红门捷径         │
│ 红门 ← 红钥匙（在地图左下角）         │
│  ↓ 否则绕远路，可能HP不足            │
│ 起点                                │
└─────────────────────────────────────┘

大脑规划：
"为了到终点，需要蓝钥匙"
"为了快速拿蓝钥匙，最好先拿红钥匙走捷径"
"拿红钥匙的路径是..."
"执行序列：[拿红钥匙] → [过红门] → [拿蓝钥匙] → [过蓝门] → [终点]"
```

## 四、学习机制：从试错到顿悟

### 阶段1：无知（小脑主导）
```
狐狸看到敌人 → 本能躲开（小脑模式："敌人=危险"）
结果：永远到不了终点
世界模型学习："直接走 → 被阻挡"
```

### 阶段2：偶然发现（探索）
```
偶然踩到平台 → 尖刺掉落 → 敌人死亡
世界模型学习："平台 + 时间 → 敌人死亡"
记忆："平台位置"与"机关效果"关联
```

### 阶段3：主动应用（大脑主导）
```
看到敌人 + 上方有平台 → 主动规划
想象轨迹："踩平台 → 尖刺掉 → 敌人死 → 路通"
评估：短期风险可接受，长期收益确定
执行：反直觉地靠近敌人
结果：成功
```

### 阶段4：模式泛化（元学习）
```
识别模式："敌人 + 上方平台/尖刺 = 可击杀"
下次遇到类似场景，自动应用
甚至主动寻找这种组合（工具使用意识）
```

## 五、可视化设计：让"思考"可见

### 1. 子目标显示
```
狐狸头顶气泡：
┌────────────────────────────┐
│ 🎯 当前：获取血包 (+30HP)   │
│ 📋 队列：[血包]→[终点]      │
└────────────────────────────┘
```

### 2. 想象画面预览
```
侧边面板显示大脑想象：
┌────────────────────────────────┐
│ 如果我左移：                    │
│ [🦊→][尖刺][空][血包]          │
│ 预测：HP-10，到达血包位置       │
│ 评估：可接受 ✓                  │
├────────────────────────────────┤
│ 如果我右移：                    │
│ [🦊][空][敌人][终点]           │
│ 预测：HP-30，死亡               │
│ 评估：危险 ✗                    │
├────────────────────────────────┤
│ 决策：选择左移                  │
└────────────────────────────────┘
```

### 3. 顿悟时刻标记
```
当狐狸第一次主动利用尖刺击杀敌人时：
屏幕显示：💡 "学会了：平台可以触发机关！"
大脑可视化显示新建立的因果链
```
