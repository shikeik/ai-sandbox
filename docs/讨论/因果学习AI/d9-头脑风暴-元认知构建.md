## 2. 目标导向的修正：人类指令 vs 机器谓词

**原偏差**：假设用户会输入 `at(agent,3,0)` 这类谓词公式。  
**正确理解**：人类不会这样说话。人类会说：

> "你去 (2,2)"

**关键差异**：
- **绝对坐标**：`(2,2)` 是一个与当前位置无关的绝对目标
- **自然语言/意图**：指令的核心是"让 agent 的位置处于坐标 (2,2)"
- **AI 需要做的第一件事是语义解析**：把人类指令转化为内部目标表征

这意味着 d9 的入口不应该是 raw predicate，而应该是一个**目标解析层（Goal Parser）**，负责把自然语言/绝对坐标转化为 AI 的内部目标状态。

---

## 3. 学习导向的修正：元认知构建 vs 预编码规则

**原偏差**：AI 已经知道"移动会改变坐标"，只是在不同格子间验证规则。  
**正确理解**：AI 应该从**一无所知**开始，像婴儿一样通过互动构建对世界的认知。

### 元认知构建的典型案例

**场景 1：第一次移动**
> AI 收到指令"去 (2,2)"。它不知道"去"是什么意思，但它可以尝试"移动"行为。  
> 它执行"上"，发现 `agent.y` 从 0 变成了 1。  
> **知识点**："移动"可以让"坐标"产生变化。

**场景 2：撞墙**
> AI 再次执行"上"，但这次 `agent.y` 没有变。  
> 它观察到：同样的行为（上），不同的前置状态（上次 y+1 是空气，这次 y+1 是墙），导致了不同的结果。  
> **知识点**：`s(cell(x, y+1)=air) + a(up) → s'(agent.y+1)`，否则 `s' = s`。  
> 这不是一条预编码的规则，而是从经验中抽象出的**因果模型**。

**场景 3：概念涌现**
> 经过足够多的互动，AI 会自发发现：
> - 有一种东西会阻止移动（后来人类叫它"墙"）
> - 有一种行为可以改变 agent 与某种物体的关系（后来叫"拾取"、"开门"）
> - 有一种策略叫"绕路"：当直接路径被阻止时，可以通过其他坐标抵达目标

**核心洞察**：AI 不是在匹配预定义规则，而是在**构建一个世界模型（World Model）**。

## 4. 三层认知架构（靠近终极愿景的路径）

为了把"元认知构建"这个宏大目标拆解为可执行的步骤，我们将其分为三层：

### Layer 1: 感知-因果层（Perception-Causal Layer）
**功能**：观察世界，建立 `state × action → state'` 的因果映射。  
**输出**：原始经验条，例如：
```
{ before: {agent:(0,0), view:[...]}, action: "上", after: {agent:(0,1), view:[...]} }
{ before: {agent:(0,1), view:[...]}, action: "上", after: {agent:(0,1), view:[...]} }
```
**关键能力**：识别"什么变化了"和"什么没变"。

### Layer 2: 概念抽象层（Concept Formation Layer）
**功能**：从原始经验中抽象出可复用的概念和不变量。  
**输出**：
- **对象概念**："agent"、"障碍物"、"可拾取物"、"可交互物"
- **关系概念**："在...旁边"、"持有..."、"被阻挡"
- **动作概念**："移动"（改变坐标）、"拾取"（ inventory 变化）、"使用"（门状态变化）
- **因果规则**：带有抽象变量的规则，而非具体坐标的规则

**例子**：
```
IF agent at (x,y) AND cell(x, y-1) is walkable THEN "上" → agent at (x, y-1)
```
这不是针对 (0,0) 的规则，而是**泛化**到任意 (x,y) 的规则。

### Layer 3: 策略-规划层（Strategy-Planning Layer）
**功能**：基于世界模型和目标，生成行动计划。  
**输出**：动作序列、子目标分解、应急策略（绕路、试错）。  
**关键能力**：
- 目标导向的反向推理（Means-End Analysis）
- 子目标分解（"去 (2,2)" → "先离开当前墙" → "绕到右侧通道"）
- 失败时的模型修正（"我以为这条路通，但它不通，更新我的地图"）

## 5. 从"预编码规则"到"世界模型"：关键转变

| 维度 | 当前（d8） | 终极目标 |
|------|-----------|---------|
| 知识来源 | 用户演示 + 预编码规则 | 自主探索 + 因果归纳 |
| 规则形式 | 具体谓词（`cell_empty(1,0)`） | 抽象变量（`cell(x,y+1)=walkable`） |
| 规划方式 | BFS 正搜已知规则 | Means-End + 模型推演 |
| 失败处理 | 报"无计划" | 反思→实验→更新模型→重试 |
| 概念定义 | 人类预定义（钥匙、门、墙） | AI 自主涌现（通过聚类/差异分析） |
| 指令形式 | `at(agent,3,0)` | "你去 (2,2)" / "捡起钥匙开门" |

## 6. d9 的务实路径：不是一步登天，而是逐步逼近

我们承认"完全元认知构建"距离当前状态还有十万八千里。d9 的任务不是直接实现终极愿景，而是**向它靠近**。

### 阶段 A：结构化经验与世界模型雏形（d9 前半）
1. **经验结构化**：把当前 `ExperienceDB` 中的字符串经验升级为结构化的 `Transition` 对象
2. **差异分析引擎**：自动比较 `before` 和 `after`，提取"变化集"和"不变集"
3. **规则泛化**：把具体坐标规则升级为带变量的规则模板（从 `cell_empty(1,0)` 到 `cell_empty(x+1,y)`）
4. **简单目标解析器**：支持 `"去 (2,2)"` → `goal: {agent.pos: (2,2)}`

### 阶段 B：主动探索循环（d9 后半）
1. **知识缺口分析**：规划失败时，分析是世界模型缺了哪条因果边
2. **主动实验设计**：为了验证某条假设，设计最小实验（如"向右走一步看看是不是空的"）
3. **自主循环**：`接到目标 → 尝试规划 → 发现缺口 → 设计实验 → 执行 → 学习 → 重试`
4. **CLI 新指令**：`做 <目标>` 触发完整自主循环

### 阶段 C：概念萌芽（d9 之后，长期）
1. **对象发现**：通过观察状态变化，识别"哪些格子是静态的、哪些是动态的"
2. **属性聚类**：通过相似性把对象聚类为"墙类"、"可拾取类"、"门类"
3. **自然语言桥接**：把人类词汇（"钥匙"、"门"）与 AI 自发聚类的概念进行对齐

## 7. 待讨论的技术问题

1. **规则的泛化表示**：如何用 TypeScript 表示带变量的规则？是字符串模板还是 AST？
2. **世界模型 vs 规则库**：世界模型是否应该取代当前的 `RuleDB`，还是作为 `RuleDB` 的上层抽象？
